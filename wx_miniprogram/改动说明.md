# 晾衣架 & 窗户控制页 — 新增「童锁」与「定时自动关闭」改动说明

> 适用文件：`pages/clothes/clothes.js`、`pages/clothes/clothes.wxml`、`pages/clothes/clothes.wxss`
> 改动原则：在原有功能完全不变的基础上新增两个功能；原有按钮的非锁定态行为、样式均未改变。

---

## 一、功能概述

1. **童锁**
   - 开启后所有控制功能（衣架/窗户按钮、定时设定、定时自动关闭）全部失效。
   - 原有 4 个控制按钮视觉置灰且不可点击；时间选择器置灰禁用。
   - 开启/关闭童锁时，向设备上报一条 `{target, action}` 格式的 JSON 指令（与伸出衣架指令同构）。
2. **定时自动关闭**
   - 用户自选时间（精确到小时:分钟），到点后自动收回晾衣架并关闭窗户。
   - 可随时取消已设定的定时任务。

### 3. sendCommand 入口新增童锁拦截（唯一对原有方法的改动）
童锁关闭时直接跳过，原有 4 个控制按钮行为完全不变；开启时拦截所有指令。
```js
async sendCommand(target, action, label) {
  // ========== 童锁拦截：开启后所有控制指令失效 ==========
  if (this.data.childLock) {
    wx.showToast({ title: '童锁已开启，操作被禁止', icon: 'none', duration: 1500 });
    return;
  }

  this.setData({ /* ... 原有逻辑不变 ... */ });
}
```

### 4. onUnload 新增定时器清理
```js
onUnload() {
  this._manualDisconnect = true;
  if (this.data.warningTimer) clearTimeout(this.data.warningTimer);
  // 清理未执行的定时任务
  if (this._timerHandle) {
    clearTimeout(this._timerHandle);
    this._timerHandle = null;
  }
  // 不主动断开全局连接
}
```

### 5. 新增方法（文件末尾）
| 方法 | 作用 |
|------|------|
| `onToggleChildLock(e)` | 切换童锁；开启时自动取消未执行的定时任务；并上报童锁状态 JSON |
| `onSetTimer(e)` | 接收 `<picker mode="time">` 的 `e.detail.value`，校验后调用 `scheduleClose` |
| `scheduleClose(hour, minute)` | 计算延迟（今天已过则顺延次日），到点触发 `executeScheduledClose` |
| `executeScheduledClose()` | 依次发送 `hanger/retract` + `window/retract`（受童锁约束） |
| `onClearTimer()` | 取消已设定的定时任务 |

```js
// ========== 新增功能一：童锁 ==========
onToggleChildLock(e) {
  const locked = e && typeof e.detail === 'object' && e.detail !== null && e.detail.value !== undefined
    ? !!e.detail.value
    : !this.data.childLock;

  this.setData({ childLock: locked });

  if (locked) {
    // 开启童锁时，取消尚未执行的定时任务
    if (this._timerHandle) {
      clearTimeout(this._timerHandle);
      this._timerHandle = null;
    }
    this.setData({ timerEnabled: false, timerText: '' });
    wx.showToast({ title: '童锁已开启', icon: 'none', duration: 1500 });
  } else {
    wx.showToast({ title: '童锁已关闭', icon: 'none', duration: 1500 });
  }

  // 上报童锁状态到设备（与伸出衣架 {target, action} 同格式）
  // 注意：直接 publish，绕过 sendCommand 的童锁拦截，否则开启后会被自身挡住
  if (this.mqtt && this.upTopic) {
    this.mqtt.publish(this.upTopic, { target: 'lock', action: locked ? 'lock' : 'unlock' })
      .then(() => console.log('[Clothes] 童锁状态已上报:', locked ? 'lock' : 'unlock'))
      .catch(err => console.error('[Clothes] 童锁状态上报失败:', err));
  }
},

// ========== 新增功能二：定时自动关闭 ==========
onSetTimer(e) {
  if (this.data.childLock) {
    wx.showToast({ title: '童锁已开启，无法设置定时', icon: 'none', duration: 1500 });
    return;
  }
  let value = (e && e.detail && typeof e.detail.value === 'string') ? e.detail.value : null;
  if (!value) {
    value = `${this.data.timerHour}:${this.data.timerMinute}`;
  }
  const parts = value.split(':');
  if (parts.length < 2) {
    wx.showToast({ title: '时间格式错误', icon: 'none', duration: 1500 });
    return;
  }
  const hour = parseInt(parts[0], 10);
  const minute = parseInt(parts[1], 10);
  if (isNaN(hour) || isNaN(minute) || hour < 0 || hour > 23 || minute < 0 || minute > 59) {
    wx.showToast({ title: '时间超出范围', icon: 'none', duration: 1500 });
    return;
  }
  this.scheduleClose(hour, minute);
},

scheduleClose(hour, minute) {
  if (this._timerHandle) {
    clearTimeout(this._timerHandle);
    this._timerHandle = null;
  }

  const now = new Date();
  const target = new Date();
  target.setHours(hour, minute, 0, 0);
  if (target.getTime() <= now.getTime()) {
    target.setDate(target.getDate() + 1);
  }
  const delay = target.getTime() - now.getTime();
  const text = `${String(hour).padStart(2, '0')}:${String(minute).padStart(2, '0')}`;

  this._timerHandle = setTimeout(() => {
    this._timerHandle = null;
    this.setData({ timerEnabled: false, timerText: '' });
    this.executeScheduledClose();
  }, delay);

  this.setData({
    timerEnabled: true,
    timerHour: String(hour).padStart(2, '0'),
    timerMinute: String(minute).padStart(2, '0'),
    timerText: text
  });

  wx.showToast({ title: `已设定 ${text} 自动关闭`, icon: 'none', duration: 1500 });
},

executeScheduledClose() {
  Promise.resolve()
    .then(() => this.sendCommand('hanger', 'retract', '定时收衣架'))
    .then(() => this.sendCommand('window', 'retract', '定时关窗'))
    .then(() => {
      wx.showToast({ title: '定时任务：已关闭衣架和窗户', icon: 'none', duration: 2000 });
    })
    .catch((err) => {
      console.error('[Clothes] 定时关闭执行失败:', err);
    });
},

onClearTimer() {
  if (this._timerHandle) {
    clearTimeout(this._timerHandle);
    this._timerHandle = null;
  }
  this.setData({ timerEnabled: false, timerText: '' });
  wx.showToast({ title: '已取消定时', icon: 'none', duration: 1500 });
}
```

## 五、行为总结

| 场景 | 表现 |
|------|------|
| 童锁关闭 | 所有功能正常，按钮彩色可点 |
| 童锁开启 | 4 个控制按钮灰显且不可点；时间选择器置灰禁用；`sendCommand` 拦截所有指令并提示 |
| 童锁切换 | 向设备上报 `{target:'lock', action:'lock'/'unlock'}`（直接 publish，绕过拦截） |
| 设定定时 | 到点自动收回衣架 + 关闭窗户（走 `executeScheduledClose`） |
| 定时中开启童锁 | 定时任务被取消，且到点指令也会被拦截 |
| 页面卸载 | 清理未执行定时句柄，避免泄漏 |

## 六、已知限制 / 待确认

- **定时可靠性**：定时基于页面 `setTimeout`，小程序被系统回收后定时丢失；如需可靠跨进程定时需在端侧/云端实现（按需求未补充端侧代码）。
- **童锁指令字段**：`target:'lock'`、`action:'lock'/'unlock'` 为按衣架指令风格拟定，若设备端协议约定其他取值（如 `target:'childlock'`、`action:'on'/'off'`），需按实际协议调整。
- **窗户状态显示**：仍为「乐观更新」（界面状态依据点击按钮直接写入，未使用设备下行 `window` 反馈），此为原有逻辑，本次未改动。
